1. Deployment

This page explains how to deploy a distribution of Key2Gym.

1.1. Java

This program is written in Java. Therefore, it requires Java Runtime Environment to be available at the target computer. Go to Java website for instructions on how to install Java Runtime Environment on your target OS.

1.2. Database

Key2Gym uses external Relational Database Management System (RDBMS) to store its business data. As for now the only supported RDBMS is PostgreSQL. The program has been intensively tested with PostgreSQL 9.1, however the later version should work as well. Go to PostgreSQL website to obtain a distribution of PostgreSQL server. If you are running Linux/Unix/BSD, we suggest you to install the PostgreSQL server from the official repository. See the PostgreSQL documentation, if you need help with installing or configuring the server.

The following is a typical setup of a user and a database: 

postgres=# CREATE USER myuser PASSWORD 'mypassword';
postgres=# CREATE DATABASE mydatabase OWNER myuser;
postgres=# \c mydatabase
postgres=# ALTER SCHEMA public OWNER TO myuser;

Note that you also need to modify pg_hba.conf to accept TCP/IP connections from the hosts, where the application is going to be used. For example, if the application is going to be run from the same host as the server, you need to have: 

host     mydatabase     myuser     127.0.0.1/32    password

The application will automatically create the schema on the database you specify in the connection properties file (see Configuration page). When you update the application, it will also update the schema on the first launch without any data being lost, as well.

1.3. Key2Gym

  1. Unzip the distribution archive to the target folder.
  2. Configure the application as explained on the Configuration page.
  3. You can now run the program with java -jar key2gym.jar.
