2.2.0 - 3.0.0

  • Client: A fresh install is of the client is recommended. Make sure you keep the logs files, if you need them. The format of etc/application.properties and connection properties is now almost completely different. Make sure you configure the client properly.
  • Database: The support of MySQL database is dropped. It's been replaced with PostgreSQL. Check Database Installation page for instructions on how to setup the PostgreSQL server. In order to migrate you need to perform the following steps:

    • Excecute the following commands on the database: 
      UPDATE order_line_orl SET `iddsc_orl` = NULL WHERE `iddsc_orl` = 0;
      
DELETE FROM `session_ssn`;
    • Generate a compatible dump: 
      mysqldump --skip-opt --no-create-info --compatible=postgresql,ansi --skip-quote-names --default-character-set=utf8 --complete-insert database --tables client_cln ad_source_ads client_profile_cpf administrator_adm client_freeze_cfz key_key attendance_atd item_itm time_split_tsp item_subscription_its discount_dsc order_ord order_line_orl property_pty session_ssn cash_adjustment_cad
    • Create a new database in PostgreSQL and migrate to version 1. Note that the migration is now done using a standalone tool. See Database Schema page for details.
    • Import the generated dump: psql -f mysql_dump.sql
    • Migrate to version 6.
    • You now have to update sequences to match just imported data. We provide you this handy script to help with that: 
      CREATE OR REPLACE FUNCTION reset_sequence(tablename text, columnname text, sequence_name text)
RETURNS void AS
$BODY$
DECLARE
BEGIN

EXECUTE 'SELECT setval( ''' || sequence_name  || ''', ' || '(SELECT MAX(' || columnname || ') FROM ' || tablename || ')' || '+1)';

END;

$BODY$
LANGUAGE plpgsql;
       
select column_name || '_seq', reset_sequence(table_name, column_name, column_name || '_seq') from information_schema.columns where column_default like 'nextval%';
    • Time splits now have titles that are displayed in appropriate places. You need to set them manually in the time_split_tsp table.
  • A brand new piece of software is now required: OpenEJB server. See Business section for details.
