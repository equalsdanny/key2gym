2. Configuration

This page explains how to configure Key2Gym.

2.1. Client

The client's configuration is located in etc/application.properties. The following is the table of supported properties:
Key Description Required
locale.language The application locale's language. Yes
locale.country The application locale's country Yes
ui The Java L&F to use in the application. It's recommended to use Nimbus as the application's L&F. Yes
connection No, but must be specified either in the properties file or as a command-line argument.
The following is the table of supported locales:
Language Country Description
en US English (United States)
ru RU Russian (Russia)

2.2. Connections

Connections describe how the application should connect to the OpenEJB server. Connections are stored as files in etc/connections/. A connection file's name has the following format: <name>.properties. A connection file itself is a Java-style properties file.

The following is the table of supported properties:

Key Description Required
connection.title The string that is going to displayed in the application as the description of this connection. The value can by any string. Note that you have to escape the Unicode characters. See the properties format description for details. No
connection.url The URL of the OpenEJB server to connect to. Yes
The connection to use has to be specified either in the application.properties file or as the command-line argument.

2.3. Logging

Key2Gym uses Log4J as its logging provider. The logging configuration file is located in etc/logging.properties and is a Log4J logging properties file. You can find the documentation on this file here.
