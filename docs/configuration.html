2. Configuration

This page explains how to configure Key2Gym.

2.1. Application

The application configuration is located in etc/application.properties. The following is the table of supported properties:
Key Description Required
locale.language The application locale's language. Yes
locale.country The application locale's country Yes
ui The Java L&F to use in the application. It's recommended to use Nimbus as the application's L&F. Yes
The following is the table of supported locales:
Language Country Description
en US English (United States)
ru RU Russian (Russia)

2.2. Connections

Connections describe how the application should connect to the storage that is to be used. Connections are stored as files in etc/connections/. A connection file's name has the following format: <codeName>.properties. A connection file itself is a Java-style properties file.

The following is the table of supported properties:

Key Description Required
type Defines the type of this connection. Built-in types of connections: PostgreSQLEclipseLink. Other connections can be supported by adding new classes at runtime. See ConnectionsManager class in the developer's documentation. Yes
title The string that is going to displayed in the application as the description of this connection. The value can by any string. Note that you have to escape the Unicode characters. See the properties format description for details. Yes
All other keys are type-specific.

PostgreSQLEclipseLink is the only built-in connection type as for now. It's recommended to use it with all PostgreSQL servers. The following is the table of properties specific to this connection type:

Key Description Required
host The server's host name, which usuall is a FQDN or IP address of the server. The whole range of valid values is environment-specific. Yes
port The server's port number, which can be any number in the range of 1-65535. Default port for MySQL servers is 3306. Yes
user The PostgreSQL user. The user must have permissions to create, alter and remove tables, as well as select, update and delete rows in the target database. Yes
password The PostgreSQL user's password. The password has to be plain text. Yes
database The PostgreSQL database to use. The database must already exist. The application will automatically create the schema. If the schema's version is older than the application's version, the application will apply patches to update it without any data loss. Yes
By default, the application will show a connections chooser dialog upon start up. If you want to always use the same connection, specify connection command-line argument. See Launching page for detailts.

2.3. Logging

Key2Gym uses Log4J as its logging provider. The logging configuration file is located in etc/logging.properties and is a Log4J logging properties file. You can find the documentation on this file here.
