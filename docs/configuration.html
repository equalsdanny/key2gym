This page explains how to configure Key2Gym.

Connections

Connections describe how the application should connect to the storage that is to be used. Connections are stored as files in etc/connections/. A connection file's name has the following format: <codeName>.properties. A connection file itself is a Java-style properties file.

The following is the table of supported properties:

Key Description Required
type Defines the type of this connection. Built-in types of connections: MySQLEclipseLink. Other connections can be supported by adding new classes at runtime. Yes
title The string that is going to displayed in the application as the description of this connection. Any string. Note that you have to escape the Unicode characters. See the properties format description for details. Yes
All other keys are type-specific.

MySQLEclipseLink is the only built-in connection type as for now. It's recommended to use with all MySQL serves. The following is the table of properties specific to this connection type:

Key Description Required
host The server's host name, which usuall is a FQDN or IP address of the server. The whole range of valid values is environment-specific. Yes
port The server's port number, which can be any number in the range of 1-65535. Default port for MySQL servers is 3306. Yes
user The MySQL user. The user must have permissions to create, alter and remove tables, as well as select, update and delete rows in the target database. Yes
password The MySQL user's password. The password has to be plain text. Yes
database The MySQL database to use. The database must already exist. See ddl property to see what options are, if the database does not have the proper schema. Yes
ddl Defines the tables generation strategy. If the values is create-tables, the application will create only missing tables. If the values is drop-and-create-tables, the application will drop and create new tables each time the connection is openned. No
